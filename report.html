Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 3 - Training and Testing¶

  • What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

Hint: Think about how overfitting or underfitting is contingent upon how splits on data is done.

Answer:

We need to determine the relative performance of various machine learning models, and a way we do this is to estimate the performance of these models on out-of-sample data. If we train on our entire dataset, we run the risk of overfitting our data with a too-complex model that won't be able to generalize well.

If we split our data into two sets — training and testing — our testing accuracy will be a better estimate of out-of-sample performance. We are essentially shelving a bit of our training data (in this case 20% of it) as 'testing' data that can act as out-of-sample data once we've training our model on the other 80%.

The variance of which data is put into our testing set can significantly change testing accuracy, however. Using k-fold cross validation is one way to solve this problem.

Question 7 - Grid Search¶

  • What is the grid search technique?
  • How it can be applied to optimize a learning algorithm?

Hint: When explaining the Grid Search technique, be sure to touch upon why it is used, what the 'grid' entails and what the end goal of this method is. To solidify your answer, you can also give an example of a parameter in a model that can be optimized using this approach.

Answer:

Grid Search is a technique for sweeping through the hyperparameters of various learning algorithms to determine their optimal values. Hyperparameters are determined prior to training (i.e. the max_depth of a tree or the degree of a polynomial regression) as opposed to other parameters that are determined via training.

The traditional way to perform Grid Search is to do a sweep through the hyperparameters of a given learning model. The Grid Search algorithm is guided by some metric of performance, which can for example be a K-fold cross validation of the training set. Grid Search will exhaustively try every combination of manually selected hyperparameters to determine their best values. This is the 'grid', the table or Cartesian Product of all manually determined values of the hyperparameters.

As an example of a hyperparameter that could be optimized using Grid Search would be the learning rate $\alpha$ for a Gradient Descent learning algorithm or the C and $\sigma$ hyperparameters for SVMs.

Answer:

The K-Fold cross-validation training technique is in a sense a solution to a problem itself raised by a solution to a problem.

The initial problem is that of training and testing a learning model on the same data, which is a methodological mistake. Such a model would 'overfit' the training data and be unable to well predict unseen data. As such the 'solution' to the initial problem is to hold out part of the available data as a test set. Oftentimes to avoid overfitting on the test set itself, which is a risk, yet another part of the data is held out as a 'validation set'. Training proceeds as follows:

  • training the model on the training set
  • evaluation is done on the validation set
  • when the model is optimized, final evaluation can be done on the test set.

However, a second 'problem' arises from partitioning the data into these three sets. This problem is twofold: (1) we can possibly (depending on the size of the training set) drastically reduce the number of samples which can be used for learning the model, and (2) the results can depend on a particular random choice for the pair of train/validation sets.

The 'solution' to this second problem is K-fold Cross-validation. The training set is split into 'k' smaller sets, and the procedure is as follows:

  • a model is used with k-1 of the folds as training data
  • the resulting model is validated on the remaining (untrained) fold of the data, which acts as the test set.

This process is repeated k times, using each fold as the test set for one iteration. The performance measure reported by k-fold CV (cross validation) is then the average of the values computed in the loop.

K-fold CV can be beneficial when using Grid Search to optimize a model. Recall that Grid Search is a way to select the best of a family of models, parametrized by a grid of hyperparameters. However, Grid Search risks fitting a model to training/validation/test sets of data that themselves may be the product of a random sub-grouping. As such the results of Grid Search may be biased based on the random generated separation of the data into the three sub-groups. from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV
import sklearn

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(n_splits = 10, test_size = 0.20, train_size=None, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = { 'max_depth':list(range(1,11)) }
    

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)
    
    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(estimator=regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
